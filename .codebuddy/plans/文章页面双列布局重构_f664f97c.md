## 产品概述

重新设计文章页面布局，将现有的三列网格布局改为左右双列结构，左侧展示文章列表，右侧展示分类、标签、热门文章等辅助内容，优化页面视觉层次和用户体验。

## 核心功能

- 左右双列布局：左侧70%宽度展示文章列表，右侧30%宽度展示侧边栏内容
- 文章列表使用PostCard组件展示，保留点赞、收藏等交互功能
- 右侧边栏包含：分类导航、热门标签云、热门文章推荐、网站统计、友情链接
- 响应式设计：移动端单列堆叠，平板和桌面双列显示
- 保留现有搜索、筛选、排序和分页功能
- 玻璃态设计风格，深色/浅色主题支持

## 技术栈

- 前端框架：Next.js 16 + React 18 + TypeScript
- 样式方案：Tailwind CSS
- UI组件库：Radix UI
- 动画库：Framer Motion + GSAP
- 图标库：Lucide React

## 实现方案

### 系统架构

采用响应式网格布局系统，主内容区使用单列PostCard布局，侧边栏采用卡片堆叠结构。通过CSS Grid和Flexbox实现自适应布局，移动端自动切换为单列堆叠模式。

### 核心设计决策

1. **布局策略**：使用 `grid-cols-1 lg:grid-cols-12` 网格，主内容区占 `lg:col-span-8`，侧边栏占 `lg:col-span-4`，保证内容优先级和可读性
2. **组件复用**：最大程度复用现有PostCard、TagCloud、FriendLinks等组件，减少重复代码
3. **性能优化**：使用虚拟滚动处理大量文章，懒加载图片，避免不必要的重渲染
4. **主题一致性**：所有新组件遵循现有的玻璃态设计系统和CSS变量规范

### 数据流

文章数据 → 分页处理 → 排序筛选 → PostCard渲染
分类/标签数据 → 侧边栏卡片 → 点击筛选 → 文章列表更新

### 关键实现说明

- 侧边栏组件使用GlassCard统一包裹，保持视觉一致性
- 所有交互（分类筛选、标签点击）通过URL参数状态同步，支持直接分享链接
- 移动端侧边栏内容折叠为可展开的抽屉或下拉菜单

### 性能考虑

- 热门文章从当前文章列表中按view_count排序获取，无需额外API调用
- 侧边栏组件使用React.memo优化，避免父组件重渲染导致的不必要更新
- 图片使用Next.js Image组件优化加载

## 设计风格

采用现代玻璃态设计（Glass Morphism）与科技感风格相结合，通过半透明背景、模糊效果、微妙阴影和渐变色彩营造视觉层次。整体布局清晰，主次分明，色彩对比度高，确保良好的可读性和视觉吸引力。

## 页面布局

### 左侧主内容区（70%宽度）

- **搜索筛选栏**：顶部固定，包含搜索框、排序按钮、筛选标签
- **文章列表**：单列PostCard布局，每篇文章独立卡片，间距适中
- **分页组件**：底部居中，清晰的页码导航

### 右侧边栏（30%宽度）

- **分类导航**：卡片式设计，分类名称+文章数，点击高亮
- **热门标签云**：标签大小根据文章数量动态调整，鼠标悬停放大
- **热门文章**：精简卡片列表，仅显示标题、阅读量、点赞数
- **网站统计**：4个小型统计卡片横向排列，带图标
- **友情链接**：2列网格布局，图标+名称+描述

### 响应式布局

- 移动端（<768px）：单列堆叠，侧边栏内容折叠为可展开区域
- 平板（768px-1024px）：双列布局，边栏宽度调整为35%
- 桌面（>1024px）：标准双列，主内容70% + 边栏30%

## 交互细节

- 卡片悬停：轻微上浮+阴影增强+边框发光
- 按钮点击：涟漪效果+颜色渐变
- 分类/标签筛选：实时更新文章列表，带平滑过渡动画
- 加载状态：骨架屏展示，提升感知性能
