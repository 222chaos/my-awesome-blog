## API 连接失败修复计划

### 🎯 问题根源
1. **后端服务器未运行**：端口 8989 上没有 FastAPI 服务响应
2. **CORS 配置缺失**：后端未允许 `http://localhost:3001` 的跨域请求

---

### 📋 修复步骤

#### 步骤 1：启动后端服务器
- 在后端目录运行启动命令
- 确认后端在 `127.0.0.1:8989` 正常监听
- 查看启动日志确认服务状态

#### 步骤 2：更新 CORS 配置
- 修改 `backend/.env` 中的 `BACKEND_CORS_ORIGINS`
- 添加 `http://localhost:3001` 到允许列表
- 配置为：`["http://localhost:3000", "http://localhost:3001", "http://localhost:8000"]`

#### 步骤 3：验证连接
- 访问前端留言页面测试 API 调用
- 检查浏览器控制台确认无 CORS 错误
- 验证留言数据正常加载

---

### 📁 需要修改的文件

1. **backend/.env** - 更新 CORS 配置
2. **启动后端服务** - 在终端中运行启动命令

---

### ✅ 预期结果
- 后端服务器正常运行在端口 8989
- 前端可以成功调用留言 API
- 浏览器控制台无 CORS 错误
- 留言和弹幕数据正常显示
