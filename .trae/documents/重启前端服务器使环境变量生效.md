## 修复 Timeline API 错误

**问题诊断**:

1. **前端需要重启**: `.env.local` 的更改需要重启 Next.js 开发服务器才能生效
2. **API 路径正确性**: 前端配置现在正确指向 `http://localhost:8989/timeline-events`

**修复步骤**:

### 步骤 1: 重启前端开发服务器
停止当前运行的前端服务器（端口 3000）并重新启动：
```powershell
# 在 frontend 目录
npm run dev
```

### 步骤 2: 验证后端服务器运行
确认后端服务器（端口 8989）正常运行，可以通过浏览器访问：
```
http://127.0.0.1:8989/api/v1/timeline-events/
```

### 步骤 3: 检查数据库中是否有 timeline_events 数据
如果数据库中没有 timeline_events 数据，需要运行 seed_test_data.py：
```powershell
# 在 backend 目录
.venv\Scripts\python scripts\seed_test_data.py
```

### 预期结果
- 前端重新加载后使用正确的 API_BASE_URL
- `timelineService.getTimelineEvents()` 请求 `http://localhost:8989/api/v1/timeline-events/` ✅
- Timeline 组件正常显示时间轴数据

**注意**: 由于 `.env.local` 已更改为 `http://localhost:8989`（移除了 `/api/v1`），前端服务器的重启是必须的。
