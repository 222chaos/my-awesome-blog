## 修复登录 API 404 错误

### 问题原因
前端 API 配置不一致，导致登录请求路径错误：
- 实际请求：`http://localhost:8989/auth/login` (404)
- 正确路径：`http://localhost:8989/api/v1/auth/login`

### 修复方案

1. **统一 API 配置**
   - 修改 [config/api.ts](file:///e:\project\my-awesome-blog\frontend\src\config\api.ts#L4) 中的 `API_BASE_URL` 添加 `/api/v1`
   - 修改 [.env.local](file:///e:\project\my-awesome-blog\frontend\.env.local#L1) 中的 `NEXT_PUBLIC_API_URL` 添加 `/api/v1`

2. **移除重复配置**
   - [auth.ts](file:///e:\project\my-awesome-blog\frontend\src\lib\api\auth.ts#L14) 中移除本地的 `API_BASE_URL`，改用 `@/config/api`

3. **更新 profile.ts**
   - [profile.ts](file:///e:\project\my-awesome-blog\frontend\src\lib\api\profile.ts#L18) 中移除本地的 `baseUrl`，改用 `@/config/api`

4. **重启前端开发服务器**
   - 停止当前的 `npm run dev`
   - 重新启动以加载新的环境变量

### 预期结果
- 所有 API 请求使用统一的基准 URL
- 登录功能正常工作
