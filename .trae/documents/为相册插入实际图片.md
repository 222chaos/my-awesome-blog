## 计划概述

创建一个脚本，为现有的相册添加实际图片数据，解决前端显示相册图片的问题。

## 当前问题

1. **数据库中已有相册数据** - Portfolio表有22个相册记录
2. **但缺少图片关联** - 没有实际的Image记录
3. **没有PortfolioImage关联** - 相册与图片之间没有关联关系
4. **前端API返回空数据** - `/api/v1/albums/{id}/images` 返回空数组或仅有封面图

## 解决方案

创建 `scripts/seed/seed_album_images.py` 脚本：

### 功能设计

1. **创建Image记录** - 使用Unsplash高质量图片URL
2. **建立PortfolioImage关联** - 将图片关联到对应相册
3. **设置封面图** - 每个相册第一张图片设为封面
4. **排序设置** - 每张图片设置sort_order

### 技术实现

```python
# 图片数据结构
portfolio_images = {
    "modern-architecture": [
        {"url": "https://...", "caption": "...", "sort_order": 0, "is_cover": True},
        {"url": "https://...", "caption": "...", "sort_order": 1, "is_cover": False},
        # ... 更多图片
    ],
    "nature-landscapes": [...],
    # ... 其他相册
}

# 执行步骤：
1. 连接数据库
2. 遍历每个相册
3. 创建Image记录
4. 创建PortfolioImage关联
5. 提交事务
```

### 图片规划

为22个相册每个添加6-10张图片：
- 城市夜景、自然风光、人物肖像、旅行记忆、动物世界、美食摄影
- 建筑摄影、街头摄影、微距摄影、黑白摄影、时尚摄影
- 运动摄影、抽象摄影、航空摄影、静物摄影、水下摄影
- 花卉摄影、极简主义、星空摄影、城市人文、森林摄影、沙漠风光

**预计创建约150-200张图片记录**

## 验证方式

1. 运行脚本后，查询数据库确认Image记录创建
2. 调用 `/api/v1/albums/{id}/images` 验证图片列表
3. 前端相册页面应能正常显示图片网格
