# 修复相册页面主题切换闪烁问题

## 问题分析

用户反馈：刚进入页面时字体是白色的，之后就变成了黑色。这是典型的主题切换闪烁问题。

## 可能原因

1. **主题初始化延迟**
   - `useThemedClasses` hook 依赖 `resolvedTheme`
   - `resolvedTheme` 可能延迟更新，导致初始渲染时使用错误的主题

2. **isMounted 状态问题**
   - 虽然相册页面是客户端组件，但可能存在初始状态不一致

3. **mounted 属性未使用**
   - 相册页面的组件没有使用 `mounted` 状态来判断是否已挂载

## 解决方案

### 方案 1：添加 mounted 状态（推荐）
- 在 `AlbumsPage` 中添加 `mounted` 状态
- 使用 `mounted` 状态控制初始渲染
- 只在 `mounted` 为 true 时才显示内容

### 方案 2：改进 useThemedClasses 初始化
- 确保主题类在组件挂载前就正确设置
- 避免使用未初始化的主题值

## 实施步骤

1. **修改 AlbumsPage.tsx**
   - 添加 `mounted` 状态（使用 `useEffect` 设置）
   - 在渲染前检查 `mounted` 状态
   - 或者直接使用 `useThemedClasses` 而不依赖额外的 mounted 状态

2. **测试验证**
   - 确认页面加载时文字颜色正确
   - 确认主题切换时没有闪烁
