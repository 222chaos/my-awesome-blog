## 问题分析

前端消息页面显示"暂无信号..."（空状态），虽然 API 返回 `[]` 空数组。可能的原因：

1. **数据库为空** - 数据库中没有消息记录
2. **数据过滤问题** - 后端查询可能过滤掉了所有消息
3. **数据类型不匹配** - 后端返回格式与前端期望不符

## 排查方案

1. 检查数据库中是否有消息数据
2. 检查 API 响应格式是否正确
3. 如果数据库为空，创建测试数据
4. 验证前端能正确显示数据

## 修复步骤

1. 连接数据库检查 messages 表数据
2. 如果没有数据，插入测试消息
3. 验证 API 返回正确数据
4. 刷新前端页面确认数据显示
